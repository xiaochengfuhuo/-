# 人工智能入门学习指南

## 文档说明

这是一份面向人工智能初学者的**学习执行指南**，由原《入门文档》整理并补充而成。它不是某一门课程的完整教材，而是帮助你解决以下四个问题：

1. **从哪里开始：** 根据现有基础选择合适的起点，避免重复学习或盲目跳级。
2. **按什么顺序学：** 先掌握 Python 和公共基础，再进入计算机视觉、Transformer 或生成式 AI 等方向。
3. **每一课怎么学：** 按“看要求、学内容、跑代码、做练习、留产出、再复盘”的流程形成学习闭环。
4. **怎样判断学会了：** 以能解释、能运行、能修改和能完成练习为标准，而不是只看视频进度。

### 适用对象

- **零基础学习者：** 从“7 天 Python 基础”开始，再进入人工智能主线课程。
- **有一定基础的学习者：** 先阅读课程文档和路线图，自主选择起始阶段。
- **不确定自己水平的学习者：** 先让智能体完成定级测试，再按推荐路线学习。

### 建议阅读顺序

1. 先在“确定学习起点”中选择自己的入口。
2. 再阅读“总体学习路线”，了解各阶段之间的关系。
3. 按对应阶段开始学习，并对每一课执行标准流程。
4. 使用大模型、笔记模板和验收清单辅助学习与复盘。

> **核心原则：先判断起点，再沿主线学习；每一课都完成“看、读、跑、练、测、复盘”闭环。**

## 一、开始之前：确定你的学习起点

先根据自己的情况选择入口：

| 当前情况                                  | 建议入口                                 |
| ----------------------------------------- | ---------------------------------------- |
| 没有编程基础                              | 从“第三部分：7 天 Python 基础”开始       |
| 会写 Python，但不熟悉 Git、环境配置或调试 | 先完成主线课程的 `00 配置与工具链`       |
| 已有数学、机器学习或深度学习基础          | 阅读路线图自行选择，或先做智能体定级测试 |
| 不确定掌握程度                            | 优先使用智能体完成十道题定级测试         |

### 方法一：根据文档自主选择

先阅读[课程主页](https://github.com/fancyboi999/ai-engineering-from-scratch-zh)、[课程路线图](https://github.com/fancyboi999/ai-engineering-from-scratch-zh/blob/main/ROADMAP.md)和[前置知识说明](https://aieng-zh.cn/prereqs.html)，检查自己是否已经掌握各阶段的前置知识：

- 已经熟悉 Python、Git 和基础开发环境，可以快速检查或跳过重复的工具入门内容。
- 已经掌握线性代数、微积分和概率基础，可以测试自己是否能够跳过数学基础阶段。
- 已经学过机器学习，但对反向传播、损失函数等不熟悉，建议从深度学习核心阶段开始。
- 如果跳级后发现代码或概念难以理解，应及时回到对应的前置课程补课。

### 方法二：让智能体测试水平（推荐）

如果使用的智能体已经安装本课程提供的 skill，可以直接输入：

```text
/find-your-level
```

智能体会通过十道题评估当前知识水平，将结果映射到合适的起始阶段，并生成包含预计课时的个性化学习路线。每完成一个阶段，还可以继续输入：

```text
/check-understanding 3
```

其中 `3` 表示要检查的阶段编号，可以替换为其他阶段。

> **注意：** `/find-your-level` 和 `/check-understanding` 是课程 skill 提供的命令，只有安装并支持该 skill 的客户端才能直接执行。在普通 AI 助手中输入这些命令，可能只会被当作普通文本；此时请使用下方的通用定级提示词。

## 二、总体学习路线

| 阶段                | 主要任务                                         | 完成标志                                   |
| ------------------- | ------------------------------------------------ | ------------------------------------------ |
| 起点判断            | 自主对照课程要求，或完成智能体定级测试           | 明确起始阶段和需要补齐的前置知识           |
| Python 基础（按需） | 零基础或基础薄弱者用约 7 天掌握基础语法          | 能独立完成小练习，并看懂基础 Python 代码   |
| AI 公共基础         | 完成配置、数学、机器学习和深度学习核心内容       | 能解释主要概念，运行并修改课程代码         |
| 方向学习            | 根据目标学习 Transformer、生成式 AI 或计算机视觉 | 完成对应课程练习，留下代码、笔记和阶段总结 |

完整的学习逻辑是：

```text
判断当前水平
      ↓
按需补齐 Python，并检查开发工具
      ↓
数学基础 → 机器学习基础 → 深度学习核心
                              ├── 计算机视觉
                              └── Transformer → 生成式 AI
      ↓
每课完成代码、练习、产出和复盘
      ↓
通过阶段验收后进入下一阶段
```

## 五、每一课的标准学习流程

<img src="images\image-20260714113209948.png" alt="image-20260714113209948" style="zoom:50%;" />

### 第 1 步：看清课程要求

开始前先确认：

- 前置课程是否已经掌握；
- 本课预计用时和学习目标；
- 本课属于 `Learn` 还是 `Build`；
- 需要运行哪些代码、提交哪些练习和产出。

预计用时只用于排期。初学者超过预计时间很正常，关键是确认时间花在理解、调试和练习上，而不是停留在被动观看。

### 第 2 步：视频与文档结合学习

1. 先快速浏览标题、目标和问题背景。
2. 观看讲解视频，记录不理解的关键词和公式。
3. 阅读课程文档，补齐视频中略过的推导和细节。
4. 用自己的话写出本课的核心结论，避免直接复制原文。

### 第 3 步：真正运行代码

- 找到该课程目录中的 `code/` 文件。
- 先原样运行，确认环境和依赖正常。
- 逐段阅读，给关键变量、函数和数据流加注释。
- 修改一个参数或实现，观察结果如何变化。
- 遇到报错时，保留报错信息和解决过程，形成调试笔记。
- <img src="images\image-20260714113256295.png" alt="image-20260714113256295" style="zoom:50%;" />

### 第 4 步：完成练习和测验

学习完成后，必须完成课程目录中的练习、课后测验和要求的产出。不要只看答案：先独立思考并写出自己的实现，再对照资料检查。

### 第 5 步：检查本课产出

课程常见目录结构如下：

```text
phases/<阶段>/<课程>/
├── code/       # 可运行代码
├── docs/       # 课程文档
└── outputs/    # 本课产出的提示词、skill、agent 等
```

完成后至少保留：

- 一份可运行的代码；
- 一页学习笔记；
- 练习或测验结果；
- 一个“仍不理解的问题”清单；
- 一段 3～5 句的课程总结。

### 第 6 步：课后复盘

每课结束后回答以下问题：

1. 我今天真正学会了什么？
2. 哪个概念我能不看资料讲清楚？
3. 哪段代码我能独立重写？
4. 练习是否全部完成？错误原因是什么？
5. 还有什么没有理解，下一步如何解决？

## 六、如何正确使用大模型辅助学习

看不懂时可以询问大模型，但目标是获得解释、提示和检查，而不是直接代写答案。可使用下面的提问模板：

```text
我正在学习【课程/主题】。

我目前理解的是：【用自己的话描述】
我不理解的是：【指出公式、概念、代码或报错】
相关代码/报错：【粘贴最小必要片段】

请你：
1. 先用直观例子解释；
2. 再逐步解释原理；
3. 给一个最小可运行示例；
4. 不直接给课后题最终答案，只给提示；
5. 最后用 3 个问题检查我是否真正理解。
```

使用大模型后，必须自己重新表述结论，并独立运行或重写代码。涉及公式、API 和运行结果时，应以课程材料、官方文档和实际执行结果为准。

## 八、阶段性验收与复习

完成一个阶段后，不要只以“视频看完”为标准。至少满足以下条件：

- [ ] 能用自己的话解释本阶段的核心概念。
- [ ] 能运行所有重点代码，并说明输入、处理过程和输出。
- [ ] 能独立重写至少一个关键实现。
- [ ] 课程练习和测验已经完成。
- [ ] 对错误和调试过程有记录。
- [ ] 整理出一份阶段总结和待复习清单。
- [ ] 知道下一阶段为什么需要这些前置知识。

建议每天学习结束后用 10 分钟复盘，每周再用 30～60 分钟整理错题、代码和未解决问题。
